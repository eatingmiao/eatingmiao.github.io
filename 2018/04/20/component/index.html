Web Component

发表于 2018-04-20   |  

观望这几年的前端框架，React、Vue、Angular2+ 在应用打包，状态管理，响应性和CLIs等方面正在互相学习和渗透，最终选用的解决方案也大同小异。

React 为现代前端开发普及了 Comonent 的概念，Vue 一开始就将组件作为其核心特性之一。在 Angular v1.x 时期，社区就有人开始编写基于组件的指令，后来 Angular v1.6 引入了 angular.component() 以便于编写基于组件的指令。2016年发布的 Angular 2.0 才为开发者带来真正的组件。

4月18日在美国盐湖城举办的 ng-conf 会议，Angular 核心开发人员之一 Rob Wormald 也表示，在 Angular v6 及以后的迭代中也依然沿用 Web Component 作为核心元素。

定义

Web components are a set of web platform APIs that allow you to create new custom, reusable, encapsulated HTML tags to use in web pages and web apps. Custom components and widgets build on the Web Component standards, will work across modern browsers, and can be used with any JavaScript library or framework that works with HTML.

Web 组件是一套基于 Web 平台的标准化API，允许开发者在 Web 页面和 Web 应用程序中创建新的自定义、可重用、封装的 HTML 标记。在Web组件标准上构建的自定义组件和小部件将在现代浏览器中运行，并且可以与任何需要使用 HTML 的 Js 库或框架一起使用。

Web Component 由四种主要技术组成：

  • Custom Elements
  • Shadow DOM
  • HTML Templates
  • HTML Imports

通过 Custom Elements 自定义标签元素来引入 HTML Templates 是可重用组件化的基础。Shadow DOM 可以隔离组件间的冲突和影响，HTML Imports 提供了一种将一个.html文档中定义的所有资源导入另一个文档的范围的方法，该方法不能有效控制管理资源的依赖性而被 Mozilla 反对。

Custom Elements

该项标准还处于non-normative的状态，请查看最新的规范。

Custom Elements 提供了一些回调函数控制自定义元素的生命周期：

  • connectedCallback：在自定义元素首次连接到文档的DOM时调用。
  • disconnectedCallback：在自定义元素与文档的DOM断开连接时调用。
  • adoptedCallback：在自定义元素移动到新文档时调用。
  • attributeChangedCallback：在添加，删除或更改其中一个自定义元素的属性时调用。

封装一个 hello-world 组件，该组件绑定一个允许更改的 name 属性，并对其值进行监听：

class HelloWorld extends HTMLElement {
    static observedAttributes = ['name']
    attributeChangedCallback(key, oldValue, newValue){}
    connectedCallback(){}
    disconnectedCallback(){}
    set name(value){}
    get name(){}
    onClick(){
        this.dispatchEvent(new CustomEvent('nameChange'),{...}));
    }

}
customElements.define('hello-world', HelloWorld);

Shadow DOM

Shadow DOM 为 Web 组件中的 DOM 和 CSS 提供了封装。因为组件的样式与主文档的样式也可能会互相影响，而实际在使用组件的时候并不需要关心组件的内部结构，只需关心组件本身，所以 Shadow DOM 把组件元素与文档隔离和封装，能更好地组织页面元素。

Shadow DOM 必须附加在一个元素上，可以是HTML文件中的一个元素，也可以是脚本中创建的元素；可以是原生的元素，如\、\；也可以是自定义元素如 \。

// 创建 shadow DOM
var shadow = document.querySelector('#hostElement').attachShadow({mode: 'open'});
// 给 shadow DOM 添加文字
shadow.innerHTML = '<p>Here is some new text</p>';
// 添加CSS，将文字变红
shadow.innerHTML += '<style>p { color: red; }</style>';

实现

React、Vue、Angular2+ 三个框架都可以在组件中指定事件处理器并对 DOM 事件进行监听和触发。以下用三种方式封装一个记录点击次数的按钮。

在 Angular 中自定义事件可以通过调用 EventEmitter 的 emit 方法来触发：

@Component({
  selector: 'counter',
  template: `
   <button (click)="increment()">
        
   </button>`,
})
class CounterComponent {  
  private count: number = 0;
  @Output() increment = new EventEmitter<number>();

    increment() {
      this.count += 1;
      this.increment.emit('increment');
    }
}

在 Vue 中可以通过注射 $emit 来触发自定义事件：

Vue.component('counter', {  
  template: `
   <button @click="increment">
        
   </button>`,
  data: function () {
    return { count: 0 };
  },
  methods: {
    increment() {
      this.count += 1;
      this.$emit('increment');
    },
  },
})

React 实现自定义事件的方式有点不同，Angular 和 Vue 作为子组件的事件会向父组件传递，而 React 是让子组件继承父组件的 props，props 就相当于 Angular 的 inputs，依据父组件传递给 Counter 的数据流，当 state 改变时，Counter 触发 increment 事件进行重新渲染。

class Counter extends React.Component { 
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  increment() {
    this.setState(
      prevState => ({ count: prevState.count + 1 }),
      () => {
        this.props.handleIncrement(this.state.count);
      },
    );
  }

  render() {
    return (
      <button onClick={() => this.increment()}>
        { this.state.count }
      </button>
      );
  }
}
